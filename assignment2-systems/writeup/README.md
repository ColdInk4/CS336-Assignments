# Writeup

The Typst entry point is `main.typ`. Each problem lives in `problems/P*.typ`.
Figures used by the writeup should live in `assets/` if needed.

This PDF tracks the written, benchmarking, accounting, and leaderboard deliverables from the handout. Code-only implementation problems are left out of the writeup body.

To build the PDF on a machine with Typst installed:

```sh
typst compile writeup/main.typ writeup.pdf
```

Keep the generated `writeup.pdf` at the repository root when preparing the Gradescope submission.
